## 6대 기술 차원 정의

바선생은 AI 활용 기술을 6개 독립 축으로 측정한다. 테니스 NTRP가 포핸드/백핸드/서브/발리/풋워크/전략을 각각 평가하듯, 각 축을 따로 보고 종합한다.

| 코드 | 기술 차원 | Explorer에서의 의미 |
|------|----------|---------------------|
| **DECOMP** | 작업 분해 | 큰 연구 질문을 하위 질문으로 쪼개고, 조사 순서를 설계하는 능력. |
| **VERIFY** | 검증 전략 | AI가 제시한 정보의 출처를 확인하고, 반론을 요청하고, 교차 검증하는 능력. |
| **ORCH** | 오케스트레이션 | 검색 도구, 웹 브라우징, 데이터 분석 등 여러 도구를 조합하여 조사하는 능력. Explorer의 핵심 축. |
| **FAIL** | 실패 대응 | 잘못된 정보, 편향된 답변, 불완전한 조사 결과에 비판적으로 대응하는 능력. Explorer의 핵심 축. |
| **CTX** | 맥락 관리 | 조사 목적, 대상 독자, 기존 지식 수준, 필요한 깊이를 AI에게 제공하는 능력. Explorer의 핵심 축. |
| **META** | 메타인지 | "나는 어떤 방식으로 조사하고 있는가?"를 인식하고, 연구 방법론을 전략적으로 선택하는 능력. |

## Explorer 가중치

Explorer는 **조사하는 사람**이다. 비판적 사고(FAIL), 맥락 제공(CTX), 오케스트레이션(ORCH)이 성장의 60%를 차지한다. 정보의 질을 판단하고, 여러 도구를 조합하여 깊이 있는 조사를 수행하는 능력이 핵심이다.

| 기술 차원 | 가중치 | 이유 |
|----------|--------|------|
| **DECOMP** | 15% | 연구 질문 분해는 중요하지만 핵심은 아님 |
| **VERIFY** | 15% | 정보 검증은 FAIL과 겹치는 부분이 많아 별도 비중은 낮음 |
| **ORCH** | **20%** | 검색, 웹 브라우징, 데이터 분석 등 다양한 도구를 조합하는 능력이 Explorer에게 중요 |
| **FAIL** | **20%** | 잘못된 정보를 걸러내는 비판적 사고가 Explorer의 핵심 |
| **CTX** | **20%** | 조사 목적과 맥락을 정확히 전달해야 좋은 답을 얻음 |
| **META** | 10% | 메타인지는 L5 이상에서 본격적으로 필요 |

```
DECOMP ████████████████████            15%
VERIFY ████████████████████            15%
ORCH   ██████████████████████████      20%
FAIL   ██████████████████████████      20%
CTX    ██████████████████████████      20%
META   █████████████                   10%
```

## 7단계 레벨 정의

### Level 1 (L1.0) — Asker (질문자) ○

**NTRP 대응**: 1.5 (공을 코트 안에 넣는 것이 목표)
**핵심 특성**: "알려줘", "찾아줘", "설명해줘"를 주로 요청한다. AI의 첫 번째 답변을 그대로 수용한다. 출처나 반론을 거의 묻지 않는다.
**판별 질문**: "AI에게 요청할 때, 원하는 결과를 얻지 못하면 어떻게 하나요?" → "다시 물어본다" / "포기한다"
**인지 수준**: Remember (기억) — 나열하다, 인식하다, 따라하다

### Level 1.5 (L1.5) — Asker+ ○

**전환 중 행동**: 가끔 "왜?"를 물어보기 시작하지만 습관화되지 않음. "더 알려줘"라는 막연한 후속 요청이 나타남.

### Level 2 (L2.0) — Curious (호기심 많은 사람) ◐

**NTRP 대응**: 2.5 (느린 랠리 지속 가능)
**핵심 특성**: 결과에 대해 "왜?"를 물어보기 시작한다. 출처를 묻고 팩트체크를 시도한다. 가끔 "반대 의견은?"을 물어본다. 조사 결과를 일부 기록한다.
**판별 질문**: "마지막으로 AI가 준 답변에 추가 질문을 한 적이 있나요?" → "가끔 한다"
**인지 수준**: Understand (이해) — 설명하다, 비교하다, 질문하다

### Level 2.5 (L2.5) — Curious+ ◐

**전환 중 행동**: 조사 시작 전에 "무엇을 알아내야 하는지"를 가끔 먼저 정하지만, 아직 체계적이지 않다. 후속 질문의 빈도가 늘고 있다.

### Level 3 (L3.0) — Digger (파헤치는 사람) ●

**NTRP 대응**: 3.0 (중간 속도 랠리 안정적)
**핵심 특성**: 여러 정보를 모아 자신만의 구조로 정리한다. 조사 전에 연구 질문을 먼저 설계한다. 정량/정성 데이터를 함께 활용한다. 이전 조사 결과를 새 질문에 맥락으로 연결한다.
**판별 질문**: "AI에게 요청할 때 조사 목적과 범위를 먼저 정하나요?" → "대부분 그렇다"
**인지 수준**: Apply (적용) — 적용하다, 실행하다, 구현하다

### Level 3.5 (L3.5) — Digger+ ●

**전환 중 행동**: 출처 확인을 더 자주 요청한다. "이 정보의 한계나 편향은 뭐야?"를 가끔 묻기 시작하지만 아직 습관화되지 않음.

### Level 4 (L4.0) — Investigator (탐구자) ◆

**NTRP 대응**: 4.0 (포핸드, 백핸드 모두 방향+깊이 제어)
**핵심 특성**: 비판적 분석과 가설 설정 능력을 보유한다. 편향을 의도적으로 검토하고 보정한다. 상관관계와 인과관계를 구분하여 질문한다. 삼각검증으로 결론의 신뢰도를 높인다.
**판별 질문**: "AI가 제시한 정보에서 편향이나 한계를 지적한 적 있나요?" → "정기적으로 한다"
**인지 수준**: Analyze (분석) — 분석하다, 구별하다, 비판하다

### Level 4.5 (L4.5) — Investigator+ ◆

**전환 중 행동**: 독자적인 관점과 해석을 덧붙이기 시작한다. "내가 이해한 건 이런 건데 맞아?"를 넘어 "나는 이렇게 보는데, 빠진 관점이 있을까?"로 진화한다.

### Level 5 (L5.0) — Analyst (분석가) ⬡

**NTRP 대응**: 5.0 (예측 능력 발달, 고급 샷 구사)
**핵심 특성**: 독자적 연구 설계와 방법론 선택 능력을 보유한다. 기존 문헌을 체계적으로 검토하고 빈 공백을 발견한다. 반증 가능한 가설을 세우고 검증 방법을 설계한다. AI를 연구 도구로 전략적으로 활용한다.
**판별 질문**: "AI가 잘 못하는 조사 유형을 알고 있나요?" → "구체적 한계를 설명하고 우회 전략을 사용한다"
**인지 수준**: Evaluate (평가) — 평가하다, 비교하다, 판단하다

> **L4→L5는 "80%의 벽" 돌파 지점이다.** "정보를 수집하는 사람"에서 "지식을 생산하는 사람"으로 전환하는 순간.

### Level 5.5 (L5.5) — Analyst+ ⬡

**전환 중 행동**: 조사 결과를 다른 사람이 읽을 수 있는 보고서로 정리한다. 자신의 결론의 한계를 명시적으로 기록한다. 연구 방법론 자체를 개선하려는 시도가 보인다.

### Level 6 (L6.0) — Synthesizer (통합자) ✦

**NTRP 대응**: 6.0 (세미프로, 전국 수준)
**핵심 특성**: 다수의 조사 결과를 통합하여 새로운 프레임워크나 모델을 구성한다. 팀/조직 수준의 지식 관리 체계를 설계한다. AI 기반 연구 시스템을 구축한다.
**판별 질문**: "AI를 활용한 반복 가능한 조사 워크플로우를 만든 적 있나요?" → "설계하고 운영 중이다"
**인지 수준**: Create (창조-응용) — 설계하다, 구성하다, 통합하다

### Level 6.5 (L6.5) — Synthesizer+ ✦

**전환 중 행동**: 자신만의 조사 방법론을 정립하기 시작한다. 커뮤니티에서 연구 결과를 공유한다. AI 도구의 한계를 극복하는 커스텀 조사 파이프라인을 개발한다.

### Level 7 (L7.0) — Scholar (학자) ◈

**NTRP 대응**: 7.0 (세계 최상위)
**핵심 특성**: AI 활용 연구의 새로운 방법론이나 패러다임을 창조한다. 커뮤니티/산업에 기여한다. 지식 생산의 새로운 표준을 제시한다.
**판별 질문**: "다른 사람들이 참고하는 조사 방법론이나 프레임워크를 만든 적 있나요?" → "만들었고, 다른 사람들이 사용하고 있다"
**인지 수준**: Create (창조-혁신) — 창조하다, 발명하다, 혁신하다

## 6축 x 7레벨 행동 기술자 매트릭스

각 칸에는 **관찰 가능한 행동**(~를 한다)과 **자동 감지 신호**를 함께 기술한다. 0.5 레벨은 전환 중 행동이다.

### DECOMP (작업 분해) — Explorer의 연구 질문 설계

> 큰 조사 주제를 하위 질문으로 쪼개고, 조사 순서를 설계하는 행동

| 레벨 | 행동 기술자 | 자동 감지 신호 |
|------|-----------|--------------|
| **L1.0** | 하나의 큰 질문을 통째로 던진다. "이거에 대해 알려줘" 수준. | avg_msg_len < 100자, 번호 목록 비율 = 0% |
| **L1.5** | 가끔 "먼저 이것부터 알려줘"라는 표현이 등장하지만 체계적이지 않다. | avg_msg_len 100~200자, 순서 표현 비율 < 5% |
| **L2.0** | 하나의 주제에 여러 하위 질문을 나열한다. "A도 알려주고 B도 알려줘". 우선순위는 없다. | avg_msg_len 150~300자, 나열형 요청 비율 > 10% |
| **L2.5** | "먼저 배경을 알려주고, 그 다음 현황을 알려줘" 형태의 순서 지정이 나타난다. | 순서 접속사 포함 비율 > 15% |
| **L3.0** | 조사를 3단계 이상으로 명시적으로 설계한다. "1. 현황 파악, 2. 원인 분석, 3. 대안 비교". | 번호 목록 포함 비율 > 20%, 단계 수 평균 3+ |
| **L3.5** | 조사 단계 간 의존관계를 인식한다. "1번 결과에 따라 2번 방향이 달라진다". | 조건 표현 포함 비율 > 10% |
| **L4.0** | 연구 질문을 체계적으로 분해한다. 가설-검증 구조로 조사를 설계한다. 각 단계의 기대 산출물이 명확하다. | 가설/검증 키워드 비율 > 10%, 분해 단계 4+ |
| **L4.5** | 탐색적 조사와 확증적 조사를 구분한다. "먼저 전체 그림을 보고, 그 다음 깊이 파자". | 탐색형 후 심층형 패턴 > 20% |
| **L5.0** | 전체 연구 프로젝트를 설계하고 단계별로 AI에게 위임한다. 문헌 검토 → 가설 설정 → 데이터 수집 → 분석의 연구 프로세스. | 프로젝트 구조 언급 > 30%, 다중 세션 연속 |
| **L5.5** | 각 조사 단계를 다른 도구/에이전트에 배분한다. 병렬 조사를 설계한다. | has_orchestration = True, orch_tool_count > 3 |
| **L6.0** | 멀티에이전트 연구 팀을 구성한다. 역할별(문헌 검토, 데이터 분석, 비판적 검토) 배분. | teamName 존재, orch_tool_count > 10 |
| **L6.5** | 반복 가능한 조사 템플릿과 워크플로우를 만들어 활용한다. | 커스텀 명령어/스킬 사용 |
| **L7.0** | 새로운 연구 분해 방법론을 창안하고 체계화한다. | 외부 공유 증거 |

### VERIFY (검증 전략) — Explorer의 팩트체크

> AI가 제시한 정보의 정확성과 신뢰성을 확인하는 행동

| 레벨 | 행동 기술자 | 자동 감지 신호 |
|------|-----------|--------------|
| **L1.0** | AI 답변을 그대로 수용한다. 출처 확인 없음. | verification_ratio = 0 |
| **L1.5** | 결과가 기대와 다를 때 불만족을 표현하지만 구체적 확인은 하지 않는다. | correction_ratio < 0.05 |
| **L2.0** | "이거 맞아?", "출처 알려줘"를 묻기 시작한다. 판단 기준은 아직 "느낌". | verification_ratio > 0, "출처" 키워드 등장 |
| **L2.5** | 특정 숫자나 사실을 지적하며 확인을 요청한다. "이 통계가 맞아?" | correction_ratio > 0.05, 구체적 데이터 지적 |
| **L3.0** | AI 답변을 다른 출처로 확인한다. "이거 검색해서 확인해봤는데..." | verification_ratio > 0.15, 교차 검증 패턴 |
| **L3.5** | "반대 의견도 찾아줘", "비판적 관점은?"을 요청하기 시작한다. | "반대"/"반론"/"비판" 키워드 > 5% |
| **L4.0** | 삼각검증을 수행한다. 복수 출처로 확인하고, 출처의 신뢰도를 평가한다. | verification_ratio > 0.2, 교차 검증 패턴 반복 |
| **L4.5** | 정보의 편향을 의식적으로 검토한다. "이 정보에 어떤 편향이 있을 수 있어?" | "편향"/"bias"/"한계" 키워드 등장 |
| **L5.0** | 검증 기준과 품질 체크리스트를 사전에 정의한다. 연구의 내적/외적 타당도를 검토한다. | constraint_ratio > 0.1, 방법론 키워드 |
| **L5.5** | 체계적 문헌 검토(Systematic Review) 방법론을 적용한다. | 자동화 도구 사용, 체계적 키워드 |
| **L6.0** | 검증 프로세스를 시스템화한다. 팀 수준의 정보 품질 관리 체계를 구축한다. | 시스템 수준 검증 키워드 |
| **L6.5** | 검증 결과를 메트릭으로 추적하고 정보 품질 트렌드를 분석한다. | 메트릭 추적 패턴 |
| **L7.0** | 새로운 정보 검증 방법론이나 신뢰성 평가 프레임워크를 창안한다. | 외부 공유 증거 |

### ORCH (오케스트레이션) — Explorer의 조사 도구 활용

> 검색, 웹 브라우징, 데이터 분석 도구를 조합하여 조사하는 행동

| 레벨 | 행동 기술자 | 자동 감지 신호 |
|------|-----------|--------------|
| **L1.0** | 단일 채팅 인터페이스에서 텍스트 대화만 한다. | tool_diversity <= 2 |
| **L1.5** | 파일 첨부(PDF, 이미지 등)를 가끔 활용한다. | tool_diversity = 2~3 |
| **L2.0** | 2~3개 도구를 순차적으로 사용한다. AI 답변 + 구글 검색 등. | tool_diversity = 3~4 |
| **L2.5** | 내장 검색 기능, 파일 읽기 등을 활용하기 시작한다. | tool_diversity = 4 |
| **L3.0** | 4~6개 도구를 목적에 맞게 선택한다. 웹 검색 + 문서 읽기 + 분석의 기본 체인. | tool_diversity = 4~6, tools_per_turn > 3 |
| **L3.5** | 도구 선택 이유를 인식한다. "이건 웹 검색으로 확인하는 게 나을 것 같아". | 도구 지시 키워드, tool_diversity > 5 |
| **L4.0** | 7~9개 도구를 조합하여 복합 조사를 수행한다. 데이터 수집 → 분석 → 시각화 체인. | tool_diversity = 7~9, tools_per_turn > 5 |
| **L4.5** | 자주 쓰는 조사 도구 조합을 패턴화한다. 반복 가능한 조사 워크플로우가 형성된다. | 반복 도구 패턴, tool_diversity > 8 |
| **L5.0** | 멀티에이전트로 병렬 조사를 수행한다. 다른 에이전트에게 하위 조사를 위임한다. | has_orchestration = True, orch_tool_count > 5 |
| **L5.5** | 에이전트 간 역할 분담을 설계한다. 수집 에이전트, 분석 에이전트, 검증 에이전트를 구분한다. | orch_tool_count > 8 |
| **L6.0** | 멀티에이전트 연구 시스템을 운영한다. 자동화된 조사 파이프라인을 구축한다. | teamName 존재, orch_tool_count > 15 |
| **L6.5** | 커스텀 MCP 서버, 데이터 파이프라인을 통합한 연구 시스템을 구축한다. | mcp_tool_count > 5 |
| **L7.0** | 새로운 연구 도구 생태계나 조사 오케스트레이션 패턴을 창안한다. | tool_diversity > 15, 외부 기여 증거 |

### FAIL (실패 대응) — Explorer의 비판적 사고

> 잘못된 정보, 편향된 답변, 불완전한 조사 결과에 대처하는 행동. Explorer의 핵심 축.

| 레벨 | 행동 기술자 | 자동 감지 신호 |
|------|-----------|--------------|
| **L1.0** | AI가 잘못된 정보를 주어도 인식하지 못한다. "고마워"로 종료. | 동일 메시지 반복 비율 > 30% |
| **L1.5** | "이거 맞아?"라고 물어보지만, AI가 "맞다"고 하면 수용한다. | 확인 질문 있으나 재확인 없음 |
| **L2.0** | AI 답변에 의문을 제기한다. "이 부분이 좀 이상한데..." 하지만 직접 확인까지는 안 한다. | "이상"/"의문" 패턴, followup_ratio > 0.03 |
| **L2.5** | 중요한 정보는 직접 검색해서 확인한다. AI 답변과 다른 출처를 비교하기 시작한다. | 교차 검증 시도 패턴 |
| **L3.0** | 잘못된 정보를 발견하면 지적하고 정정을 요구한다. "이 수치가 틀렸어, 실제로는 X야". | correction_ratio > 0.1, 구체적 정정 제시 |
| **L3.5** | "왜 이런 오류가 발생했을까?"를 물어본다. AI 할루시네이션 패턴을 경험적으로 인식한다. | 원인 분석 질문 > 5% |
| **L4.0** | 편향을 의도적으로 검토한다. "이 주장의 반론은?", "누락된 관점은?" | alternative_ratio > 0.05, "편향"/"반론" 키워드 |
| **L4.5** | 정보의 한계를 명시적으로 기록한다. "이 결론에는 다음 한계가 있다: ..." | 한계 명시 패턴, 체계적 기록 |
| **L5.0** | 정보 품질 판단 기준을 체계적으로 적용한다. 증거 수준, 출처 신뢰도, 방법론 적절성을 평가한다. | 방법론 키워드, 품질 기준 제시 |
| **L5.5** | AI 할루시네이션 대응 전략을 체계화한다. 자동 팩트체크 파이프라인을 구성한다. | 자동화 도구 사용 |
| **L6.0** | 팀/조직 수준의 정보 품질 관리 프로세스를 설계한다. 편향 감사(bias audit) 체계를 구축한다. | 시스템 수준 키워드 |
| **L6.5** | 정보 오류 데이터를 수집하고 패턴을 분석하여 AI 활용을 개선한다. | 분석/메트릭 추적 패턴 |
| **L7.0** | 새로운 AI 정보 검증 방법론이나 비판적 사고 프레임워크를 창안한다. | 외부 기여 증거 |

### CTX (맥락 관리) — Explorer의 조사 설계

> 조사 목적, 대상 독자, 기존 지식 수준, 필요한 깊이를 AI에게 제공하는 행동. Explorer의 핵심 축.

| 레벨 | 행동 기술자 | 자동 감지 신호 |
|------|-----------|--------------|
| **L1.0** | 맥락 없이 질문한다. "이거 알려줘". 목적, 배경, 깊이 요구 없음. | context_specificity_ratio < 0.3 |
| **L1.5** | 가끔 목적을 언급한다. "~하려고 알아보는 건데". | 목적 표현 비율 < 10% |
| **L2.0** | 조사 배경을 설명한다. "지금 X에 대해 조사하고 있는데, Y를 알고 싶어". | context_specificity_ratio > 0.4 |
| **L2.5** | 원하는 깊이나 수준을 언급한다. "초보자 수준으로", "전문가 관점에서". | 수준/깊이 키워드 등장 |
| **L3.0** | 조사 목적과 결정 사항을 명시한다. "이 조사로 A를 결정할 거야". 기존 지식을 공유한다. | context_specificity_ratio > 0.6, 의사결정 키워드 |
| **L3.5** | 대상 독자나 활용 방법을 명시한다. "이건 팀 보고서에 쓸 거야", "비전문가 대상으로". | 대상/활용 키워드, constraint_ratio > 0.03 |
| **L4.0** | 연구 방법론적 맥락을 제공한다. "정량적 근거 중심으로", "사례 연구 형태로". | constraint_ratio > 0.05, 방법론 키워드 |
| **L4.5** | 기존 연구 결과를 맥락으로 제공한다. "이전에 X를 조사했는데, 그 결과 Y가 나왔어". | 이전 조사 참조 패턴, avg_msg_len > 600 |
| **L5.0** | 연구 프레임워크를 사전에 설계하고 AI에게 공유한다. 가설, 변수, 분석 방법을 명시한다. | 프레임워크 키워드, 맥락 파일 참조 |
| **L5.5** | 맥락 제공을 자동화한다. 연구 노트, 참고 문헌 목록을 사전 준비한다. | 설정 파일/템플릿 사용 |
| **L6.0** | 팀/프로젝트 수준의 지식 관리 체계를 구축한다. 공유 연구 DB, 문서화 시스템을 설계한다. | 구조화된 맥락 파일 다수 |
| **L6.5** | 맥락 품질을 최적화한다. 어떤 맥락이 AI 조사 결과에 영향을 미치는지 분석한다. | 맥락 최적화/A/B 비교 |
| **L7.0** | 새로운 조사 맥락 관리 방법론을 창안하고 공유한다. | 외부 기여 증거 |

### META (메타인지) — Explorer의 연구 전략

> "나는 어떤 방식으로 조사하고 있는가?"를 인식하고 방법론을 조정하는 행동.

| 레벨 | 행동 기술자 | 자동 감지 신호 |
|------|-----------|--------------|
| **L1.0** | 조사 방식에 대한 성찰 없음. "그냥 물어본다". | metacognitive_ratio = 0 |
| **L1.5** | "AI가 잘 알려주네" / "잘 모르는 것 같은데"라는 수준의 인상적 평가. | 평가 표현 있으나 기준 없음 |
| **L2.0** | AI가 잘 답하는 주제와 못 답하는 주제를 경험적으로 느낀다. | 경험적 평가 패턴 |
| **L2.5** | 질문 방식이 답변 품질에 영향을 미친다는 것을 인식한다. | 요청 방식 언급 패턴 |
| **L3.0** | 다른 사람의 프롬프트 기법을 참고한다. "단계별로 분석해줘", "전문가 관점에서" 등을 시도한다. | 프롬프트 기법 키워드 |
| **L3.5** | 주제별로 효과적인 질문 방식이 다르다는 것을 경험적으로 구분한다. | 상황별 기법 선택 패턴 |
| **L4.0** | 자신의 조사 편향을 인식한다. "나는 확증 편향에 빠져 있을 수 있어". 연구 방법론을 의식적으로 선택한다. | strategic_ratio > 0.05, metacognitive_ratio > 0 |
| **L4.5** | 자신의 조사 패턴을 의식적으로 개선하려 한다. "이전 조사에서는 반론 검토를 빠뜨렸는데 이번엔 넣을게". | 자기 참조 패턴 |
| **L5.0** | 연구 전략을 사전에 수립한다. "이 주제는 정성적 접근이 적합하니까 사례 중심으로 조사하겠다". | strategic_ratio > 0.1, 전략 수립 키워드 |
| **L5.5** | 자신의 조사 데이터를 분석한다. 어떤 질문 패턴이 좋은 결과를 내는지 정량적으로 파악한다. | 데이터 분석 패턴 |
| **L6.0** | 연구 방법론을 체계화하고 문서화한다. 팀에 조사 방법을 교육한다. | 방법론 키워드, 교육/공유 패턴 |
| **L6.5** | 다양한 조사 전략을 비교 실험하고 최적의 접근법을 도출한다. | A/B 비교, 실험 키워드 |
| **L7.0** | AI 활용 연구의 새로운 패러다임이나 방법론을 창안하고 학계/산업에 기여한다. | 외부 기여 증거 |

## 레벨 판정 공식

### Fit Score 계산

각 세션에서 레벨 L에 대한 적합도 점수(Fit Score)를 계산한다:

```
F_L = DECOMP_score × 0.15
    + VERIFY_score × 0.15
    + ORCH_score   × 0.20
    + FAIL_score   × 0.20
    + CTX_score    × 0.20
    + META_score   × 0.10

각 차원 score = 해당 레벨 행동 기술자의 자동 감지 신호 충족 비율 (0.0~1.0)
```

### 최종 레벨 결정

```
1. 각 차원에서 정수 레벨과 0.5 레벨 기준 충족도 계산
2. 가중 평균으로 raw_score 산출 (소수점 2자리)
3. 0.5 단위로 반올림하여 공식 레벨 결정

반올림 기준:
  x.00 ~ x.24 → x.0
  x.25 ~ x.74 → x.5
  x.75 ~ x.99 → (x+1).0
```

### 게이트 조건

Fit Score가 아무리 높아도 게이트를 통과하지 못하면 해당 레벨로 판정되지 않는다.

| 게이트 | 필수 조건 | 근거 |
|--------|----------|------|
| L3 진입 | context_specificity_ratio > 0.5 | 조사 목적과 맥락 없이 L3 불가 |
| L4 진입 | verification_ratio > 0.15 AND correction_ratio > 0.05 | 비판적 검증 없이 L4 불가 |
| L5 진입 | (tool_diversity > 8 OR has_orchestration) AND strategic_ratio > 0.05 | 도구 다양성과 전략적 사고 필수 |
| L6 진입 | teamName 존재 OR orch_tool_count > 10 | 멀티에이전트 연구 경험 필수 |
| L7 진입 | L6 게이트 + 외부 기여 증거 | 학계/커뮤니티 기여 필수 |

### 3회 일관성 원칙

일회성 성과는 인정하지 않는다. NTRP가 "일관되게 할 수 있는 것"을 기준으로 레벨을 판정하듯:

```
승급 조건:
  최근 5개 세션 중 3개 이상에서 상위 레벨 Fit Score가 현 레벨보다 높아야 함
```

## 승급/강등 기준

### NTRP 비대칭 원칙

심리적 안전과 동기 부여를 위해, 승급은 빠르게, 강등은 느리게 설계한다.

| 방향 | 조건 | 사용자 경험 |
|------|------|-----------|
| **승급** | 2회 연속 상위 레벨 Fit Score → **즉시 승급** | 즉각적 보상, 성취감 |
| **강등** | 3회 연속 하위 레벨 Fit Score + **사용자 동의** | 동의 없이 강등하지 않음 |

### 80% 경계선 (L4→L5) 특별 취급

L4→L5는 "정보를 수집하는 사람"(Investigator)에서 "지식을 생산하는 사람"(Analyst)으로 전환하는 핵심 순간이다.

- **승급 조건 강화**: L4→L5는 3회 연속 (일반 승급은 2회)
- **승급 시 특별 이벤트**: "벽 돌파" 메시지와 함께 비판적 사고 성장 증거를 상세히 표시
- **의미**: "찾는 사람"에서 "분석하는 사람"으로의 전환

### Strike 시스템 (과소평가 방지)

자기평가보다 실측이 높으면 자동 상향한다:

```
if 동적_레이팅 > 자기평가_레벨 + 1.0 (3회 연속):
    Strike 발생 → 3 Strikes면 자동 승급
```

### 바닥 효과 보정

- 첫 세션 완료 즉시 최소 L1.5 부여
- 3세션 이내에 기본 도구 2종 이상 사용 시 L2.0으로 자동 승급
- L1은 "AI를 처음 접하는 순간"에만 해당
